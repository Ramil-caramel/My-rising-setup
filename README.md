## NIRI

лежит в _.config/niri/config.kdl_

```
layout "us,ru"
options "grp:caps_toggle,grp:win_space_toggle,grp_led:caps"
```

`layout "us,ru"` - языки используемые для ввода.

`options ""` - строка специальных XKB-флагов.

`grp:caps_toggle,grp:win_space_toggle` - переключение раскладки на CAPS LOCK и win+ space.

`grp_led:caps` - включет лампочку когда активна втор ая группа, то есть русский язык.

Что такое XKB - Система управления клавиатурой в Linux/Unix. Превращает скан-коды клавиш в символы, обрабатывает раскладки, модификаторы и индикаторы.

Как устроен XKB - Два уровня: пользовательский RMLVO (раскладки, опции) и низкоуровневый KcCGST (скан-коды, символы).

Связь с Niri - Niri не использует XKB-протокол X11. Вместо этого он применяет библиотеку libxkbcommon — нативную Wayland-реализацию XKB.

```
layout {
    // Set gaps around windows in logical pixels.
    gaps 16
    background-color "transparent"
    ...
}
```
`gaps 16` - Отступы между окнами и границами.

`background-color "transparent"` - Устанавливает цвет фона рабочего пространства (пространства за окнами) как полностью прозрачный. Чтобы кореутно отобрадались обои.

Тут просто толщина кольца подстветки активного окна

`spawn-at-startup "mpvpaper" "-p" "-o" "no-audio hwdec=auto loop panscan=1.0" "*" "~/Pictures/furina.mp4` 

`mpvpaper` — это утилита для установки живых обоев (видео) на фон рабочего стола в Wayland-композиторах.

`-p` — Pause on visibility (обычно). Ставит видео на паузу, когда все окна свернуты или когда вы переключаетесь на другое рабочее пространство.

`-o` - передает следующие параметры напрямую в плеер mpv.

`no-audio` — Отключает звук.

`hwdec=auto` — включает аппаратное декодирование видео (VA-API / VDPAU / NVDEC). Это переносит нагрузку с процессора на видеокарту, спасая батарею ноутбука и снижая нагрев.

`loop` — зацикливает воспроизведение.

`panscan=1.0` — Масштабирует видео так, чтобы оно полностью покрывало экран.

`"*"` — означает все мониторы

`"~/Pictures/furina.mp4"` - путь к файлу с анимацией
```
layer-rule {
    match namespace="^mpvpaper$"
    place-within-backdrop true
}
```
Гарантирует, что живые обои всегда будут вести себя корректно, не мешая работе с обычными окнами.

`match namespace="^mpvpaper$"` — Niri ищет окно (клиент Wayland), чьё пространство имен (namespace) в точности совпадает со строкой mpvpaper. ^ и $ — это регулярное выражение, означающее «строго от начала до конца» (защита от случайных совпадений, например, если бы существовал mpvpaper-test).


`place-within-backdrop true` — принудительно помещает это окно в слой «Задник» (Backdrop). Это самый нижний из возможных слоев рендеринга в Niri.

```
window-rule {
    geometry-corner-radius 12
    clip-to-geometry true
    
    // Отключает заливку фона цветом рамки под прозрачным окном
    draw-border-with-background false
}
```

`geometry-corner-radius 12` - Устанавливает радиус скругления углов каждого окна в 12 логических пикселей.

`clip-to-geometry true` - Включает жесткую обрезку (clipping) содержимого окна по его новой геометрии (со скруглением).

`draw-border-with-background false` - Отключает рисование непрозрачного фона под рамкой (бордюром) окна. Так как Kitty у меня прозрачен, то без команды выделение аидно.

По нажатию Super + W отправляет сигнал USR1 процессу(ам) с именем waybar через команду pkill. Что переключает его отобрвжение.

## Rofi

лежит в _.config/rofi/config.rasi_

Что такое .desktop-файл

Это стандартный конфигурационный файл, который содержит всю информацию о приложении для графической среды. В операционных системах на основе Linux он выполняет ту же роль, что и ярлык в Windows.

Где хранятся: Обычно в каталогах /usr/share/applications/ (для системных программ) и ~/.local/share/applications/ (для программ конкретного пользователя).

Структура: Это текстовый файл с расширением .desktop, который содержит набор ключей в формате Ключ=Значение.

Основные ключи:

`Type=Application` — указывает, что это приложение.

`Name=Firefox` — имя приложения, которое вы видите в меню.

`Exec=firefox %u` — команда для запуска программы.

`Icon=firefox` — путь к иконке приложения.

`Categories=Network;WebBrowser;` — категории для отображения в меню.

Назначение: Благодаря этим файлам ваша система знает, какие программы установлены, как их запускать, какие у них иконки и в каких категориях меню их показывать


```
configuration {
    modi: "drun,run";
    font: "JetBrainsMono Nerd Font 11";
    show-icons: true;
    icon-theme: "Papirus-Dark";
    drun-display-format: "{name}";
    
    // Скрываем лишние префиксы
    display-drun: "󰀻 ";
    display-run: " ";
}
```


`modi	"drun,run"`	- Указывает, какие режимы (модули) будут доступны.

• drun — сканирует .desktop-файлы и показывает приложения с иконками.

• run — позволяет вводить команды вручную (как в терминале).

Порядок определяет, какой режим активен по умолчанию (первый — drun).

`font	"JetBrainsMono Nerd Font 11"` - Задает шрифт для всего интерфейса. Nerd Font включает иконки, которые используются в display-drun и display-run.

`show-icons	true` - Включает отображение иконок приложений рядом с названиями (берутся из .desktop-файлов).

`icon-theme	"Papirus-Dark"` - Задает тему иконок, из которой Rofi будет брать изображения для приложений.	Системный — зависит от установленных тем в системе.

`drun-display-format	"{name}"` - Определяет, как отображать запись приложения в режиме drun. Здесь указано только имя (без комментариев, без описания). Можно было бы добавить {generic}, {comment} и т.д.

`display-drun	"󰀻 "`	Заменяет стандартный заголовок для режима drun (обычно "DRUN") на иконку (из Nerd Font). Позволяет визуально отличать режимы.	UI/UX — делает интерфейс более иконочным и минималистичным.

`display-run	" "`	Аналогично, для режима run — вместо текста "RUN" показывается иконка терминала.

### Доп пояснения

icon-theme "Papirus-Dark" — иконки приложений в списке

Что это:
Это настройка системного уровня, которая указывает Rofi, откуда брать иконки для каждой программы в списке.

Как это работает:
Когда Rofi показывает список приложений (режим drun), он читает .desktop-файлы и видит там строку Icon=firefox. Затем он ищет файл с таким именем в каталогах установленных тем иконок. Papirus-Dark — это одна из таких тем.

<span style="color:red">Если указанная тема не установлена в системе (например, Papirus-Dark отсутствует в /usr/share/icons/), то Rofi покажет либо пустое место, либо стандартную заглушку.</span>

display-drun "󰀻 " — иконка режима в строке ввода (промпт)

Что это:
Это настройка только для интерфейса — она меняет текст или символ, который отображается слева от поля ввода, прямо перед курсором.

Как это работает:
По умолчанию, когда Rofi запущен в режиме drun, в строке ввода вы видите что-то вроде:
text

DRUN > [поле ввода]

Или, может быть, просто >. Настройка display-drun позволяет заменить этот текст на любой другой, например, на иконку из шрифта. В вашем случае там стоит символ 󰀻 (это иконка «приложения» из Nerd Font).

Что вы увидите на экране:
Теперь слева от поля поиска будет не буквы DRUN, а маленькая иконка приложения. Например, так:
text

󰀻 [поле ввода]

То есть это просто украшение, чтобы визуально понять, что вы сейчас ищете приложения, а не вводите команду.
display-run " " — иконка режима команд (промпт для run)

Что это:
Аналогично предыдущему, но для режима run (ручной ввод команд). По умолчанию там было бы RUN >, а теперь будет иконка терминала (символ ).


```
* {
    bg:            #2e3440cc; /* Основной фон панели */
    bg-selected:   #434c5ecc; /* Фон выбранного пункта */

`*` — глобальный селектор, определяет переменные (с префиксом @) и значения по умолчанию.

`background-color: transparent;` — указывает, что по умолчанию все элементы имеют прозрачный фон (переопределяется для конкретных элементов).

`text-color: @fg;` — задаёт цвет текста по умолчанию, ссылаясь на переменную.

```
window {
    background-color: @bg;

    border-color:     #4c566a;
    border-radius:    12px;
    width:            550px;
    padding:          15px;
    
    /* Центрирование окна в Wayland */

```

`background-color	@bg`	Полупрозрачный тёмно-синий фон окна.

`border	1px`	Тонкая рамка.

`border-color	#4c566a`	Серо-синий цвет рамки (Nord).

`border-radius	12px`	Скругление углов окна — соответствует стилю Niri.

`width	550px`	Фиксированная ширина окна (не растягивается).

`padding	15px`	Внутренние отступы от краёв окна до содержимого.

`location и anchor	center`	Важно для Wayland: указывает, что окно должно быть центрировано на экране (по центру).

```
mainbox {
    children: [ inputbar, listview ];
    spacing: 10px;
}
```

`mainbox` — основной контейнер, в котором расположены два дочерних элемента: inputbar (строка ввода) и listview (список результатов).

`spacing: 10px;` — отступ между этими блоками.

```
inputbar {
    background-color: @bg-selected;
    border-radius:    8px;
    padding:          10px;
    children:         [ prompt, entry ];
}
```

`inputbar` — панель ввода. Фон чуть светлее основного, скруглённый, с внутренними отступами.

Внутри неё находятся prompt (иконка или символ приглашения) и entry (поле ввода текста).

Остальное отвечает за:

`prompt` — отображает символ (по умолчанию это > или иконка поиска).

`entry` — поле ввода.

`listview` — контейнер для списка результатов.

`element` — каждый отдельный пункт списка приложений.

`element selected ` - для настройки цвета выбранного элемента.

`element-icon` — иконка приложения внутри элемента.

`element-text` — текст названия приложения.

## Waybar

лежит в _.config/waybar/config.jsonc_ , а стили в _.config/waybar/style.css_  

```
{
  "layer": "top",
  "position": "top",
  "height": 0,
  "spacing": 4,
  "margin-top": 12,
  "margin-bottom": 5,
  "margin-left": 150,
  "margin-right": 150,
  
  "mode": "hide", 
  "exclusive": false,
}
```

`"layer": "top"` - Размещает панель в слое TOP протокола wlr-layer-shell (поверх обычных окон, но под всплывающими оверлеями).

`"position": "top"` - Физическое расположение на экране — сверху.

`"height": 0`	Автоматическая высота (подстраивается под содержимое). Если задать число, будет фиксированная.

`"spacing": 4`	Расстояние между модулями в пикселях.	

`margin-*`	12 (сверху), 5 (снизу), 150 (слева/справа)	Внешние отступы панели от краёв экрана. Слева и справа по 150px — панель не растягивается на всю ширину, а имеет «воздух» по бокам.	

`"mode": "hide"` - Панель скрыта по умолчанию (OLED-режим). Появляется при наведении мыши на край экрана или по нажатию хоткея (обычно Mod+H).	<span style="color:red">По факту не работает</span>

<span style="color:red">Я ее просто выключаю на MOD + W когда не нужно.</span>

```
  "modules-left": ["niri/workspaces"],
  "modules-center": ["clock"],
  "modules-right": [
    "tray", "network", "temperature", "cpu", "memory",

Модули сгруппированы в три области (слева, по центру, справа).
